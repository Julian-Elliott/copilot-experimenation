# GitHub Copilot Instructions

## Project Overview
This is a fake data generator application that demonstrates the use of GitHub Copilot for Python development. The project includes:

- Basic Python application structure
- Class-based code organization
- External API integration with Ollama
- CSV data generation and file handling

## Code Style Guidelines
- Use clear, descriptive function and variable names
- Include docstrings for all classes and functions
- Follow PEP 8 style guidelines
- Add type hints where appropriate
- Use meaningful comments for complex logic

## Project Structure
- `main.py` - Entry point for the application
- `fake_data_generator.py` - Core data generation class
- `create_fake_data.py` - Ollama API integration for AI-generated data
- `requirements.txt` - Python dependencies

## Development Notes
- Use virtual environments for dependency isolation
- Generate requirements.txt with `pip freeze > requirements.txt`
- Test with different data types and sizes
- Handle errors gracefully, especially for external API calls

## AI Model Integration
When working with Ollama:
- Use clear, specific prompts
- Request structured output (CSV format)
- Handle connection errors appropriately
- Provide helpful error messages for setup issues
